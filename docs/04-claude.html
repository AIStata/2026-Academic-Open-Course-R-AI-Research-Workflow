7  Claude Code 辅助科研

本章介绍如何使用 Claude Code 作为 AI 助手提升科研效率。

7.1 Claude Code 简介

Claude Code 是一个强大的 AI 编程助手，特别适合：

  • 代码编写与调试
  • 数据分析与可视化
  • 文献阅读与笔记整理
  • 论文写作辅助

7.2 安装与配置

7.2.1 安装 Claude Code

参考官方文档安装 Claude Code CLI 工具。

7.2.2 在 VS Code 中使用

在 VS Code 终端中运行：

# 启动 Claude Code
claude

7.3 常见使用场景

7.3.1 场景 1：代码调试

当代码出错时，可以向 Claude Code 提问：

问题：我的 R 代码报错了，请帮我看看问题在哪里？

library(did)
result <- att_gt(yname = "Y", gname = "group",
                idname = "id", tname = "year",
                data = mydata)

Claude Code 会分析代码并提供建议。

7.3.2 场景 2：数据可视化

问题：帮我用 ggplot2 画一个漂亮的事件研究图，显示 95% 置信区间。

示例对话：

# Claude 可能生成的示例代码
library(ggplot2)

event_study_plot <- function(data, att, se, time) {
  df <- data.frame(
    time = time,
    att = att,
    se = se
  )

  ggplot(df, aes(x = time, y = att)) +
    geom_point() +
    geom_line() +
    geom_ribbon(aes(ymin = att - 1.96 * se,
                    ymax = att + 1.96 * se),
                alpha = 0.2) +
    geom_vline(xintercept = 0, linetype = "dashed") +
    labs(x = "相对时间", y = "ATT", title = "事件研究图") +
    theme_minimal()
}

7.3.3 场景 3：表格输出

问题：帮我写一段代码，用 modelsummary 包输出回归表格到 Word。

# 示例代码
library(modelsummary)
library(fixest)

# 估计模型
model1 <- feols(Y ~ D | i + t, data = simulation)
model2 <- feols(Y ~ D + X1 + X2 | i + t, data = simulation)

# 输出表格
modelsummary(list("模型1" = model1, "模型2" = model2),
             output = "regression_table.docx",
             stars = TRUE)

7.4 实际示例：使用 Claude 辅助计量分析

让我们通过一个完整示例展示如何使用 Claude Code：

7.4.1 步骤 1：描述数据

提示词： 我有一个面板数据集，包含企业数据，变量有：企业ID、年份、 产出、投资、就业人数。请帮我写一段代码进行描述性统计。

Claude 可能生成：

# 描述性统计示例
library(dplyr)
library(tibble)

# 创建模拟数据
firm_data <- tibble(
  firm_id = rep(1:100, each = 10),
  year = rep(2010:2019, 100),
  output = rnorm(1000, 10, 2),
  invest = rnorm(1000, 2, 0.5),
  employment = sample(50:500, 1000, replace = TRUE)
)

# 描述性统计
firm_data %>%
  select(output, invest, employment) %>%
  summary()

7.4.2 步骤 2：估计模型

提示词： 请帮我用 fixest 包估计固定效应模型， 被解释变量是产出，解释变量是投资。

Claude 可能生成：

library(fixest)

# 估计模型
model_fe <- feols(output ~ invest | firm_id + year,
               data = firm_data)

# 查看结果
etable(model_fe)

7.4.3 步骤 3：生成可视化

提示词： 帮我画一个系数图，显示不同分位数回归的结果。

Claude 可能生成：

library(ggplot2)

# 假设我们有不同分位数的结果
quantile_results <- tibble(
  quantile = c(0.1, 0.25, 0.5, 0.75, 0.9),
  coef = c(0.8, 1.0, 1.2, 1.1, 0.9),
  se = c(0.2, 0.15, 0.1, 0.12, 0.18)
)

# 系数图
ggplot(quantile_results, aes(x = factor(quantile), y = coef)) +
  geom_point(size = 3) +
  geom_errorbar(aes(ymin = coef - 1.96 * se,
                    ymax = coef + 1.96 * se),
                width = 0.2) +
  geom_hline(yintercept = 0, linetype = "dashed") +
  labs(title = "分位数回归结果",
       x = "分位数", y = "系数") +
  theme_minimal()

7.5 提示词技巧

7.5.1 好的提示词要素

  1. 明确任务：清楚说明要做什么
  2. 提供背景：给出数据结构和目标
  3. 示例：给出期望输出的示例
  4. 约束条件：说明限制和要求

7.5.2 示例提示词模板

代码生成： > 我需要写一段 R 代码来 [任务描述]。 > 我的数据结构是 [描述数据]。 > 我想要的输出是 [描述期望输出]。

错误调试： > 我的代码报错了，错误信息是： > [粘贴错误信息] > 代码是： > [粘贴代码] > 请帮我找出问题并修复。

解释代码： > 请帮我解释这段代码的工作原理， > 并指出可能的改进空间。 > [粘贴代码]

7.6 注意事项

7.6.1 验证结果

Claude Code 生成的代码需要：

  1. 检查逻辑：确保代码逻辑正确
  2. 测试运行：在实际数据上测试
  3. 理解代码：确保你理解代码的每一部分
  4. 引用来源：重要方法需要查阅原始文献

7.6.2 保护数据安全

  • 不要分享敏感数据
  • 使用模拟数据来提问
  • 保密研究想法在发表前
